P.S. Чтобы заработала база данных, необходимо импортировать файл "myshop.sql".
Хост - "localhost"
Имя базы данных - "myshop"
Имя пользователя - "root"
Пароль пользователя - "root"
Или же поменять на другое в файле "function.php"
Что сделано:
    1. Поиск товаров
        1.1. Поиск не сработает, если поле пустое
        1.2. Если пользователь ввёл несуществующий товар, то появится ошибка
        1.3. Ведётся строгий режим поиска (полное название товара)
    2. Категории товаров
        2.1. Если товар данной категории отсутствует, то будет ошибка
        2.2. В случае успеха выводятся товары, соответствующие данной категории
    3. Авторизация
        3.1. Авторизация сделан через POST запрос. Теперь, если пользователь введёт неккоректные данные, то вылезет сразу ошибка
        3.2. Если пользователь авторизовался, то вместо кнопок авторизации будет его имя
    4. Личный кабинет
        4.1. Сделан вывод основной минимальной информации (имя, id пользователя, кол-во его денег)
    5. Пополнение счёта
        5.1. Пополнение счёта возможно только при успешной авторизации пользователя
        5.2. При пополнении счёта пользователь должен ввести свои данные для проверки этого пользователя в БД
            5.2.1. Если введены неккоректные данные, пользователя перенаправит на главную страницу
        5.3. Если пользователь укажет сумму пополнения больше 200 000 - появится ошибка
        5.4. Если пользователь совершает впервые пополнение счёта, то его счёт пополнится на сумму, которую он указал
        5.5. Если пользователь уже совершал пополнение счёта и имеется какой-то баланс, то произойдёт перезапись счёта с учётом денег, которые уже имелись при пополнении
        5.6. В случае успешного пополнения счёта пользователя перенаправит на главную страницу
    6. Добавление в корзину
        6.1. Пользователь не сможет добавить товар в корзину, если он не авторизован
        6.2. Для каждого пользователя хранится своя корзина в БД
        6.3. Если корзина пользователя пуста, то он сможет добавить товар в корзину, в противном случае получит уведомление
        6.4. Пользователь не сможет добавить два раза один и тот же товар в корзину
    7. Корзина
        7.1. Корзина теперь хранится не в LocalStorage, а в базе данных, так что можно смело закрывать браузер
        7.2. При удалении товара из корзины произодёт перезагрузка страницы в целях обновить содержимое корзины на стороне клиента
        7.3. Оплатить содержимое корзины можно только в том случае, если в ней есть хотя бы один товар, в противном случае невозможно попасть на страницу оплаты
        7.4. В случае удаления товара из корзины на странице оплаты, произойдёт перезагрузка страницы в целях обновить содержимое корзины на стороне клиента
    8. Оплата
        8.1. Если пользователь не пополнил счёт, но перешёл на страницу оплаты, то кнопка самой оплаты будет недоступна
        8.2. Если кол-во денег пользователя меньше кол-ва суммы итоговой корзины, то пользователь получит отказ в оплате
        8.3. Если кол-во денег пользователя больше кол-ва суммы итоговой корзины, то произойдёт оплата. При оплате будет очищена вся корзина пользователя.
    9. Внесение правок
        9.1. В админ-панеле добавлена функция перезаписи JSON файла в случае обновления товара
        9.2. Cookie-файлы обновляются при обновлении страницы
